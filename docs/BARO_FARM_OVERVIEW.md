# 바로팜(Baro-Farm) 프로젝트 개요

## 1. 프로젝트 소개

### 1.1 프로젝트 이름
**바로팜 (Baro-Farm)**

### 1.2 프로젝트 비전
신선한 농산물을 생산자로부터 소비자에게 직접 연결하는 Farm-to-Table 이커머스 플랫폼과 농장 체험 서비스를 제공하여, 도시 소비자와 농촌 생산자를 연결하는 종합 농업 플랫폼

### 1.3 판매 상품 및 서비스

#### 주요 서비스
1. **Farm-to-Table 이커머스**
   - 농가 직거래 농산물 판매
   - 신선 식품 배송 서비스
   - 계절별 농산물 큐레이션

2. **농장 체험 서비스**
   - 농장 방문 체험 예약
   - 수확 체험 프로그램
   - 농촌 관광 상품

### 1.4 타겟 사용자
- **소비자**: 신선한 농산물을 찾는 도시 거주자, 가족 단위 농장 체험 희망자
- **생산자**: 농가, 농장 운영자, 농업 법인
- **관리자**: 플랫폼 운영 관리자, CS 담당자

## 2. 주요 기능 목록

### 2.1 핵심 기능 (필수)

#### 인증/인가 (Authentication & Authorization)
- 이메일/비밀번호 기반 회원가입 및 로그인
- JWT 기반 토큰 인증
- 역할 기반 접근 제어 (RBAC): 소비자, 생산자, 관리자
- 소셜 로그인 (선택): 카카오, 네이버, 구글 (향후)

#### 업체 등록 (Farm Registration)
- 농가/농장 정보 등록 및 관리
- 사업자 등록증 검증
- 농장 프로필 관리 (위치, 소개, 이미지)
- 운영 시간 및 연락처 관리

#### 상품 등록 (Product Management)
- 농산물 상품 등록 (이름, 가격, 재고, 이미지)
- 카테고리 분류 (과일, 채소, 곡물 등)
- 계절별/시즌별 상품 관리
- 상품 옵션 관리 (무게, 패키지 단위)
- 상품 상태 관리 (판매중, 품절, 일시중지)

#### 체험 등록 (Experience Management)
- 농장 체험 프로그램 등록
- 체험 일정 및 정원 관리
- 체험 가격 및 포함 사항 설정
- 계절별 체험 프로그램 관리

#### 결제 관리 (Payment Management)
- 다중 결제 수단 지원 (카드, 계좌이체, 간편결제)
- 결제 진행 상태 관리
- 결제 내역 조회
- 환불 처리
- PG사 연동 (토스페이먼츠, 아임포트 등)

#### 정산 관리 (Settlement Management)
- 생산자별 매출 정산
- 정산 주기 관리 (주간, 월간)
- 수수료 계산 및 차감
- 정산 내역 조회 및 리포트
- 정산금 지급 처리

#### 통계 관리 (Statistics & Analytics)
- 실시간 판매 현황 대시보드
- 생산자별 매출 통계
- 인기 상품 분석
- 사용자 행동 분석
- 시간대별/일별/월별 통계

#### 검색 (Search with Elasticsearch)
- 상품명/농장명 검색
- 자동완성 기능
- 검색어 추천
- 필터링 (가격, 지역, 카테고리)
- 검색 랭킹 및 인기 검색어

#### 배송 관리 (Delivery Management)
- 주문별 배송 상태 추적
- 배송지 관리
- 택배사 연동
- 배송 완료 알림
- 배송 조회 API 제공

#### 리뷰 및 평점 시스템
- 상품 리뷰 작성 및 조회
- 별점 평가
- 포토 리뷰
- 리뷰 신고 및 관리

#### 알림 시스템 (Notification)
- 주문 상태 변경 알림
- 배송 알림
- 프로모션 및 이벤트 알림
- 푸시 알림 / 이메일 / SMS

### 2.2 확장 기능 (향후 추가)

#### AI 기반 추천 시스템
- 사용자 구매 이력 기반 상품 추천
- 협업 필터링을 통한 개인화 추천
- 계절별 추천 상품
- 유사 상품 추천

#### AI 챗봇
- 고객 문의 자동 응답
- 상품 정보 안내
- 주문 조회 및 배송 추적
- FAQ 자동 응답

### 2.3 추가 권장 기능
#### 위시리스트 (Wishlist)
- 관심 상품 저장
- 가격 변동 알림
- 재입고 알림

#### 쿠폰 및 프로모션
- 할인 쿠폰 발행
- 프로모션 코드 관리
- 이벤트 페이지
- 첫 구매 혜택

#### 구독 서비스
- 정기 배송 구독
- 제철 과일 박스 구독
- 구독 관리 (일시정지, 취소)

#### 커뮤니티
- 농장 소식 게시판
- 레시피 공유
- Q&A 게시판
- 농업 정보 공유

#### 관리자 대시보드
- 전체 시스템 모니터링
- 사용자/주문/정산 관리
- 콘텐츠 관리
- 통계 및 리포트

## 3. 기대 효과

### 3.1 생산자 관점
- 직거래를 통한 수익 증대
- 유통 마진 절감
- 농장 홍보 및 브랜딩 효과
- 안정적인 판로 확보

### 3.2 소비자 관점
- 신선한 농산물 직접 구매
- 생산자와의 신뢰 기반 거래
- 합리적인 가격
- 농장 체험을 통한 여가 활동

### 3.3 사회적 가치
- 농촌 경제 활성화
- 로컬푸드 운동 확산
- 도농 교류 증진
- 지속가능한 농업 지원

## 4. 성공 지표 (KPI)

- 월간 활성 사용자 수 (MAU)
- 주문 전환율
- 평균 주문 금액 (AOV)
- 생산자 만족도
- 고객 만족도 (CSAT)
- 재구매율
- 플랫폼 거래액 (GMV)

## 5. 리스크 및 대응 방안

### 기술적 리스크
- **MSA 복잡도**: 초기에는 모노리틱으로 시작 후 점진적 분리 고려
- **배송 연동**: 택배사 API 연동 이슈 대비 수동 관리 기능 병행
- **결제 실패**: 재시도 로직 및 롤백 전략 수립

### 비즈니스 리스크
- **생산자 확보**: 초기 파트너 농장 확보를 위한 인센티브 정책
- **배송 품질**: 신선도 유지를 위한 콜드체인 물류 파트너십
- **계절성**: 계절별 상품 다변화 및 체험 서비스로 보완

## 6. 팀 역할 분담 (5인)

### 역할별 담당 영역

| 역할 | 주요 책임 | 담당 서비스 (총 개수) | 핵심 경험 |
|-----|---------|-----------|---------|
| **🧭 PO** | 프로젝트 관리 + 개발 + 인프라 | experience, analytics + **K8s, GitHub Actions, Vercel** **(2개 + 인프라)** | Kafka, 통계, **DevOps**, **프론트** |
| **🧱 아키텍트** | MSA 설계 + 개발 | member, auth, notification, api-gateway **(4개)** | 인증/인가, Gateway, 알림, Kafka |
| **🧑‍💻 백엔드 개발자 1** | 핵심 API 개발 | farm, order, payment **(3개)** | Saga 패턴, Kafka, PG 연동 |
| **🧑‍💻 백엔드 개발자 2** | 비즈니스 로직 개발 | product, review, settlement, delivery **(4개)** | Spring Batch, 정산, 배송, Kafka |
| **🧠 AI 개발자** | AI + 검색 + 인프라 | cart, search + **AWS 관리** **(2개 + 인프라)** | Elasticsearch, Spring AI, AWS, Kafka |

### 주요 특징
- **PO**: DevOps + Front-end 담당 + 예약 (K8s, CI/CD, Vercel)
- **아키텍트**: 인증 및 게이트웨이 담당 + 알림
- **백엔드1**: 농장, 주문, 결제 (핵심 거래)
- **백엔드2**: 상품, 리뷰, 정산, 배송 (가장 많은 서비스)
- **AI**: 장바구니, 검색 + AWS 인프라

**협업 방식**: 
- 각자 담당 서비스의 Back-end 개발 (API + DB + 테스트 + 문서화)
- 아키텍트의 코드 리뷰 필수
- 주간 기술 공유 및 회고
